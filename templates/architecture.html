System Architecture

Overview

The Dual Model IDS Web System implements a two-stage attack detection pipeline with confidence-based routing. Network traffic is first classified as benign or attack, then attack type is identified. Confidence scores determine the response actions (blocking, alerting, logging).

Processing Pipeline

1
Data Upload: Network traffic CSV files are uploaded through the web interface
2
Preprocessing: Data is loaded, validated, and prepared for feature extraction
3
Stage 1 - Binary Classification: Determines if traffic is benign or attack
4
Stage 2 - Multiclass Classification: If attack detected, identifies attack type
5
Confidence Routing: Confidence scores determine response actions
6
Results Display: Visualize predictions with charts and detailed statistics

Confidence-Based Routing

The system uses three confidence levels to determine response actions:

HIGH (>90%)
  • Immediate blocking
  • SIEM alerts
  • P1 incident ticket
  • SOC notification
MEDIUM (70-90%)
  • Rate limiting
  • Analyst queue
  • Sandbox analysis
  • Threat enrichment
LOW (<70%)
  • Routine logging
  • Sampling
  • Retraining dataset
  • No impact

Machine Learning Models

Binary Classification Models

6 different models trained for benign vs. attack classification:

  • Logistic Regression: Baseline linear classifier
  • K-Nearest Neighbors: Instance-based learning
  • Naive Bayes: Probabilistic classifier
  • Decision Tree: Tree-based classifier
  • Random Forest: Ensemble method
  • Support Vector Machine: Kernel-based classifier

Multiclass Classification Models

Same 6 models trained to identify specific attack types (DoS, Probe, R2L, U2R, etc.)

Feature Engineering

PCA (Principal Component Analysis): Reduces 100+ network traffic features to 40-50 principal components (95% variance retention)

Scaling: StandardScaler normalization ensures features are on similar scales
